About Permissions
Per group permissions always override a user's default permissions. If you want users to access only those groups they've been granted specific rights to then you should leave their default permission level set to "no access."

Only administrators are exempt from this rule. Users with administrative privileges cannot be denied access to any group.

no access
The user cannot see events scheduled in this group. However, if the same event occurs in more than one group then the user will be subject to the least restrictive.
read
The user can only view events, not add or edit them.
add
The user can view and add new events, but not edit existing events.
edit
The user can view and add events as well as edit existing events.
manage
In addition to the above, the user can modify user permissions within the group.
